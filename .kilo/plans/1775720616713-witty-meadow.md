# DealRoom Custom Tab Redesign — Plan

## Status: DRAFT
## Date: 2026-04-09

---

## Purpose
Redesign the DealRoom Custom tab from a text-heavy, form-based UI into a visual-first, round-table conference metaphor with minimal reading, progressive complexity, and high visual clarity.

---

## Design Decisions (Finalized)

### 1. Layout
- **Split screen**: 70/30 left/right ratio
- Left panel = Interaction (Round Table + Actions)
- Right panel = Insights (Score, Signals, Analysis)
- Right panel **collapsible** so users can focus on table when needed

### 2. Round Table Perspective
- **Top-down bird's eye view**
- Circular table with seats positioned around perimeter
- "DEAL" or objective indicator in center
- Visual states: 🟢 aligned, 🟡 uncertain, 🔴 blocking

### 3. Stakeholder Interaction
- **Click seat → Anchored popup appears near the clicked seat**
- Spatial context preserved — popup is anchored to the seat
- Focus dimming: selected seat bright, others dimmed
- Popup content: Quote + Status + Request

### 4. Popup Content
- **Quote + Status + Request** (sweet spot)
- Quote: what the stakeholder said
- Status: 🔴 Blocking / 🟡 Uncertain / 🟢 Aligned
- Request: what they need to move forward
- On Simple: just Quote + Status. Medium+: add Request

### 5. Action Flow
- **Hybrid progressive**:
  - **Simple**: Chat-style quick action — click seat → message field appears → send
  - **Medium**: Chat + light structure (small dropdown inside chat)
  - **Hard**: Full action composer unlocked (Action Type + Target + Message + Document)
- Guided suggestions always available as optional toggle

### 6. Simulation Control
- **Primary**: "▶ Run Round" button — advances full round
- **Secondary**: Auto-play toggle with speed control (Slow/Medium/Fast)
- During auto-play: disable manual actions, show "⚡ Simulation running..."
- Optional: Pause + Skip buttons

### 7. Popup Sequencing
- **Sequential popups** — one at a time
- User clicks "Next →" to advance to next speaker
- Indicator: "Response 1 of 3"
- Optional auto-play toggle for responses

### 8. Progressive Unlock
- **Accordion with disabled sections** (not hidden, not blurred)
- All steps visible: 🟢 Simple (active) → 🔒 Medium (locked) → 🔒 Hard (locked)
- Soft preview on hover: shows what Medium/Hard introduce
- Micro-copy under locked steps explaining unlock condition
- Users see the journey ahead

### 9. Insights Panel (Right)
- **Primary (always visible)**: Score + Status
  - Score gauge (0.0–1.0)
  - Status: 🟢 Progressing / 🟡 At Risk / 🔴 Blocking
- **Secondary**: Live signals (CFO trust ↓, Alignment ↑)
- **On-demand**: Causal explanations collapsible under "▼ Why?"

### 10. Color / Typography
- Dark theme (#0d1117 background)
- **Black text (#e5e7eb on dark)** — high contrast per user requirement
- 3 text sizes only: Large (titles), Medium (labels), Small (hints)
- No long text blocks — icons + signals + short labels

## Key Technical Decisions

### State Management
- Continue using `view_state = gr.State()` — all state in single dict
- Add new keys for: `popup_queue`, `current_popup_index`, `unlocked_levels`, `show_suggestions`
- `saved_runs` continues to track completed episodes

### Round Table Rendering
- Pure CSS-positioned seats using existing `SEAT_POSITIONS` constants
- SVG not needed — CSS `position: absolute` around circular container
- Popup anchored via CSS `position: absolute` relative to seat position

### Sequential Popup Queue
- Store responses from a round in `popup_queue` list
- Track `current_popup_index` in view_state
- "Next" button increments index; wraps to end of queue

### Backend Integration
- `web_manager.reset_session()` / `step_session()` unchanged
- `_policy_action()` as baseline (used for auto-play)
- Progressive unlock: track which levels user has completed

## Success Criteria
1. Round table visual clearly shows stakeholder positions and states
2. Clicking a seat shows anchored popup with Quote + Status + Request
3. Sequential popups advance with "Next" button or auto-play
4. Score + Status always visible in right panel
5. Actions evolve from chat-style (Simple) to full composer (Hard)
6. Locked steps visible in accordion, unlocked progressively
7. **All text is black/high-contrast on dark background**
8. No console errors during normal operation
