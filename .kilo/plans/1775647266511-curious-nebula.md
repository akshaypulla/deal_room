### 5 Stakeholders with Template-Based Responses

- CFO, CTO, Legal, Procurement, Ops
- Each has 4 stances (cooperative/testing/delaying/obfuscating) ├Ś 3 satisfaction buckets (high/mid/low) = 12 response pools ├Ś 4-5 variants each
- DOCUMENT_EFFECTS dict maps document types (roi_model, security_cert, etc.) to satisfaction changes
- COLLABORATIVE_SIGNALS / AGGRESSIVE_SIGNALS for rapport computation

### Core Environment Mechanics

- **POMDP**: Agent sees noisy `engagement_level` (1-step delayed), `deal_momentum`, `veto_precursors` ŌĆö NOT absolute beliefs or satisfaction
- **Belief deltas** in `info["belief_deltas"]` ŌĆö only the *change* in beliefs, not absolute values
- **Veto risk**: Accumulates for sat < 0.30, fires at scenario-specific threshold
- **3-stage deal progression**: evaluation ŌåÆ negotiation ŌåÆ legal_review ŌåÆ final_approval ŌåÆ closed
- **CCIGrader**: Composite score at episode end (weighted consensus ├Ś implementation_risk ├Ś efficiency ŌłÆ execution_penalty)

## Known Issues Documented in plan.md (Must Preserve)

**Issue 1 ŌĆö ClaimsTracker double expansion:**
Target expansion is done by the CALLER (environment), not inside ClaimsTracker. Centralized at environment level.

**Issue 2 ŌĆö Group target belief_deltas:**
`target_responded_positively` uses `max()` across all expanded targets for group targets like "cto_cfo".

**Issue 3 ŌĆö Veto risk round 0:**
No veto accumulation on round 0 (opening observations only).

**Issue 4 ŌĆö Stage progression STAGE_MIN_ROUNDS:**
No stage advances before minimum rounds have passed. Prevents document-spam instant progression.

**Issue 5 ŌĆö Momentum three-state:**
+1 (stage advance or blocker resolved), 0 (holding), -1 (regression). Consecutive 0s signal danger.
